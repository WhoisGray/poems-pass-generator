# 🗺️ نقشه‌ی راه (Roadmap / TODO)

وضعیت فعلی پروژه پایدار است. مرحله‌ی بعدی روی **بزرگ‌کردن دیتابیس** و **آسان‌کردن افزودن شعر** تمرکز دارد.

## ✅ انجام‌شده (نسخه ۱.۰)
- [x] هسته‌ی مشترک Node + مرورگر (بدون وابستگی، آفلاین)
- [x] CLI با حالت‌های `poem` / `proverb` / `abstract`
- [x] اکستنشن Manifest V3
- [x] تصادفی امن (CSPRNG) + برآورد آنتروپی
- [x] تست‌ها و CI/CD

## 🎯 مرحله‌ی بعد: افزایش دیتابیس
- [ ] رساندن دیتابیس به **۵۰۰+ بیت شعر** و **۲۰۰+ ضرب‌المثل** (افزایش مستقیم آنتروپی)
- [ ] افزودن شاعران بیشتر (نظامی، عطار، رودکی، پروین اعتصامی، اخوان ثالث، …)
- [ ] دسته‌بندی بر اساس قالب (غزل، رباعی، مثنوی) و حال‌وهوا (عاشقانه، حماسی، عرفانی)
- [ ] افزودن وزن/طول هجا برای انتخاب هوشمندتر واژه‌ها

## 🛠️ روش‌های افزودن دیتابیس بیشتر (در دست بررسی)
- [ ] **اسکریپت ایمپورت** از فایل متنی ساده: هر بیت در یک خط → تولید خودکار `fg` با `transliterate.js`
- [ ] **ایمپورت از گنجور**: اسکریپت تبدیل خروجی [گنجور / ganjoor](https://github.com/ganjoor) به فرمت دیتابیس (به‌صورت آفلاین؛ دانلودِ یک‌بار، اجرای محلی)
- [ ] **اعتبارسنجی دیتابیس**: اسکریپت بررسی یکتایی `id`، نبودِ مدخل تکراری و درستی `fg`
- [ ] **بسته‌های دیتابیس افزونه‌ای** (data packs): امکان بارگذاری مجموعه‌های اختیاری (مثلاً «شعر کودکانه»، «حماسی»)
- [ ] ابزار CLI: `poems-pass --add` برای افزودن تعاملی بیت تازه

## 💡 ایده‌های بعدی
- [ ] انتشار رسمی در **Chrome Web Store** و **Firefox Add-ons**
- [ ] حالت PIN/عبارت‌عبور عددی فارسی
- [ ] خروجی به فرمت‌های password manager (KeePass/Bitwarden)
- [ ] رابط وب مستقل (همان هسته)

> مشارکت در بزرگ‌کردن دیتابیس بسیار خوش‌آمد است — [CONTRIBUTING.md](CONTRIBUTING.md) را ببینید.
